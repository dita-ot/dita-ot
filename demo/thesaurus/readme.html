Taxonomy Specialization

Purpose: The Taxonomy (thesaurus) specialization defines formal subjects and the relationships between those subjects so you can classify what your content is about. For a more detailed explanation, see the article at http://www-128.ibm.com/developerworks/xml/library/x-dita10/.

You can view the reference documentation for the taxonomy specialization.

Building

The Taxonomy specialization provides the conductor.xml file to extend the base Ant targets with the following build targets:

associateMap2skos
Build an association DITA map to convert a taxonomy scheme map and classification map to SKOS.
content2xhtml
Build the topic content for a taxonomy scheme map and classification map to HTML.
subjects2html
Build a catalog for a taxonomy scheme map as HTML.

The run-ant.xml file executes these build targets on the sample content. See the conductor.xml file for lower level targets.

You should run the sample Ant build from the base directory for the DITA Open Toolkit, using a relative path to the Taxonomy specialization:

ant -f demo\thesaurus\install_plugin.xml
ant -f demo\thesaurus\run_ant.xml optionalTarget

You need only execute the install_plugin.xml file after initial installation or update of any plugin.

Contents

The specialization includes subdirectories with the following contents:

doc
The source files and output for the documentation.
dtd
DTDs for the document type for the taxonomy specialization.
out
Formatted sample output files from the Ant build.
sample
Source files for an example of subjects and classified content.
xsd
XML Schemas for the document type for the taxonomy specialization.
xsl
Transforms for producing the output of the Ant build.
