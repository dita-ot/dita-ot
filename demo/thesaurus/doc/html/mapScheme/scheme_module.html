Scheme map

You define the relationships between subjects including both hierarchies and associations in the scheme map.

Description

You define a taxonomy by organizing the subjects in a hierarchy, referring to subject topics with the subjectdef element.

The scheme map also expresses associations between subjects such as use of one subject by another subject. To express associations, you use the relatedSubjects or subjectRelTable table.

Example

The following example shows a scheme that identifies the Installing and Configuring subjects as special kinds of user action subjects. The example also identifies Security and Utilization as special kinds of user concerns. 

<subjectScheme title="Sample taxonomy" id="taxonomyScheme">
    <subjectdef href="action.xml" navtitle="Action">
        <subjectdef href="installing.xml" navtitle="Installing"/>
        <subjectdef href="configuring.xml" navtitle="Configuring"/>
        ...
    <subjectdef href="concern.xml" navtitle="Concern">
        <subjectdef href="security.xml" navtitle="Security"/>
        <subjectdef href="utilization.xml" navtitle="Resource utilization"/>
        ...
  • hasInstance
    Specifies that the contained hierarchy expresses INSTANCE-OF relationships between subjects.
  • hasKind
    Specifies that the contained hierarchy expresses KIND-OF relationships between subjects.
  • hasNarrower
    Specifies that the contained hierarchy indicates a general-purpose, NARROWER-THAN relationship between subjects.
  • hasPart
    Specifies that the contained hierarchy expresses PART-OF relationships between subjects.
  • relatedSubjects
    Specifies an associative relationship for the contained subjects.
  • schemeref
    Refers to a related scheme.
  • subjAltLabel
    A synonym for the subject within the scheme.
  • subjAltSymbol
    An alternative symbol used to represent the subject within the scheme.
  • subjectdef
    Within a scheme, refers to a topic that defines a formal subject.
  • subjectHead
    Specifies a collection of subjects and assigns a title to the collection.
  • subjectmeta
    Overrides or extends the standard labels or symbols for a subject.
  • subjectRel
    Defines an association between subjects where the members of the association play different roles.
  • subjectRelHeader
    Specifies the roles played by subjects in associations.
  • subjectRelTable
    Specifies structured associations between subjects.
  • subjectRole
    Specifies a role for the contained subjects.
  • subjectScheme
    Organizes subjects into a taxonomy for processing or display.
  • subjHiddenLabel
    An name that can be applied to the subject within the scheme but shouldn't be presented to the user.
  • subjPrefLabel
    A preferred label for the subject within the scheme.
  • subjPrefSymbol
    A preferred icon for representing the subject within the scheme.
